# 미로게임 요구사항 정의서
버전 : v1.0
작성일 : 2026-03-29
작성자 : 김신영

# 1. 개요
## 1.1 프로젝트명
MVC 기반 알고리즘 선택형 미로 게임 (Maze Game for MVC)

## 1.2 프로젝트 목적
MVC(Model-View-Controller) 패턴을 준수하여 코드를 구현한다.
코드 구조를 잡는 것을 목적으로 하며 향후 Jamis Buck의 미로 생성 알고리즘(Maze Generation Algorithm Recap) 문서에 소개된 다양한 알고리즘을 학습하고, 해당 알고리즘들을 직접 테스트 및 시각화해 볼 수 있는 시스템을 구현한다.

## 1.3 프로젝트 배경
단순한 미로 찾기 게임을 넘어, 생성 알고리즘에 따라 미로의 난이도와 형태(막다른 길의 빈도 등)가 어떻게 변하는지 직관적으로 확인하고, 객체지향 설계 원칙(특히 전략 패턴)을 코드에 반영해보기 위함이다.

# 2. 시스템 범위
## 2.1 포함 범위
미로 생성 및 렌더링 로직

사용자 입력을 통한 게임 설정 (미로의 크기, 알고리즘 선택)

사용자 커서 이동 및 충돌 감지 로직

게임 승리 조건 판정 및 콘솔 화면 갱신

## 2.2 제외 범위
GUI(Graphic User Interface) 기반 프론트엔드 (본 프로젝트는 콘솔/터미널 기반으로 동작)

멀티플레이 및 네트워크 기능

데이터베이스를 활용한 사용자 기록 저장

# 3. 용어 정의
MVC: Model, View, Controller의 약자로 애플리케이션을 세 가지 역할로 구분하는 디자인 패턴.

Cell (셀): 미로를 구성하는 최소 단위 공간. 길(Path) 또는 벽(Wall)의 속성을 가짐.

Grid (그리드): 전체 미로를 구성하는 2차원 배열 구조.

Cursor (커서/캐릭터): 사용자가 조작하여 미로를 탐색하는 주체.

# 4. 기능 요구사항 (Functional Requirements)
[FR-01] 미로 크기 입력: 시스템은 게임 시작 시 사용자로부터 미로의 가로(Width) 및 세로(Height) 크기를 입력받아야 한다.

[FR-02] 단일 값 입력 처리: 사용자가 크기 값을 하나만 입력할 경우, 시스템은 가로와 세로를 동일한 값으로 상정하여 처리해야 한다.

[FR-03] 알고리즘 선택 기능: 사용자는 Jamis Buck이 소개한 알고리즘 (예: Recursive Backtracking, Prim's 등) 중 테스트할 미로 생성 방식을 메뉴에서 선택할 수 있어야 한다.

[FR-04] 미로 초기화: 생성된 미로는 명확한 시작점(Start)과 종료점(Exit)을 표시해야 하며, 사용자 커서는 시작점에 위치한 상태로 화면에 출력되어야 한다.

[FR-05] 사용자 이동 조작: 사용자는 지정된 키(W, A, S, D)를 입력하여 커서를 상, 하, 좌, 우로 이동시킬 수 있어야 한다.

[FR-06] 승리 판정 및 종료: 커서가 종료점에 도달하면 시스템은 승리 메시지를 출력하고 게임 재시작 여부를 사용자에게 물어본다.

# 5. 비기능 요구사항 (Non-Functional Requirements)
[NFR-01] 아키텍처 준수: 코드베이스는 반드시 Model, View, Controller로 분리되어야 하며, View는 Model을 직접 조작할 수 없다.

[NFR-02] 확장성 (전략 패턴): 새로운 미로 생성 알고리즘을 추가할 때, 기존 Controller나 View의 코드를 수정하지 않고 Model의 주입 방식만 변경하여 확장할 수 있어야 한다.

[NFR-03] 반응성: 사용자의 이동 키 입력 시 화면 렌더링(갱신)은 지연 없이 즉각적으로 이루어져야 한다.

# 6. 데이터 요구사항 (Data Requirements)
[DR-01] 미로 맵 구조: 미로 데이터는 메모리상에서 2차원 배열 형태로 관리되어야 하며, 벽(Wall)과 길(Path)을 구분하는 명확한 데이터를 가져야 한다.

[DR-02] 상태 데이터: 게임 시스템은 현재 커서의 좌표(x, y), 미로의 크기 정보, 현재 게임의 종료 상태(Won)를 메모리에 유지해야 한다.

# 7. 예외 처리 및 비즈니스 규칙 (Exceptions & Business Rules)
## 7.1 비즈니스 규칙 (Business Rules)
[BR-01] 최소 크기 제한: 미로의 가로와 세로 크기는 최소 10 이상이어야 한다.

[BR-02] 경로 유효성 보장: 생성된 미로는 막혀있지 않고, 시작점에서 종료점까지 도달 가능한 경로가 반드시 하나 이상 존재해야 한다.

[BR-03] 최소 통로 폭: 채워진 공간(벽) 사이의 길은 최소한 한 블록의 폭을 유지해야 한다.

[BR-04] 이동 제한: 사용자 커서는 생성된 길(Path)로만 이동이 가능하며, 벽(Wall)을 통과할 수 없다.

## 7.2 예외 처리 (Exceptions)
[ER-01] 잘못된 크기 입력: 사용자가 10 미만의 값을 입력하거나 숫자가 아닌 값을 입력할 경우, 에러 메시지를 출력하고 재입력을 요구해야 한다.

[ER-02] 잘못된 이동 입력: 사용자가 벽으로 이동하려 하거나 미로 맵 밖으로 벗어나려 할 경우, 이동을 무시하고 현재 상태를 유지해야 한다.
